# Генератор моковых данных для TaxionBack

Инструмент для заполнения базы данных тестовыми данными для разработки и демонстрации.

## Что создается

Генератор создает реалистичные моковые данные для всех основных сервисов:

### 👥 Пользователи и отделы
- 8 отделов с подразделениями
- Пользователи с полными профилями (ФИО, email, телефон, должность)
- Распределение по ролям: суперадмин, руководители отделов, админы, сотрудники
- Различные статусы пользователей (онлайн, занят, отошел, оффлайн)
- Учетные данные: `password123` для всех пользователей
- Суперадмин: `admin@taxion.ru` / `password123`

### 💬 Чаты
- Групповые чаты (60%) с 3-10 участниками
- Личные чаты (40%) между двумя пользователями
- 10-50 сообщений в каждом чате с различными типами:
  - Текстовые сообщения (80%)
  - Изображения (10%)
  - Файлы (5%)
  - Системные сообщения (5%)
- Реакции на сообщения (emoji)
- Прочитанные/непрочитанные статусы
- Избранные и закрепленные чаты

### 📋 Задачи
- Задачи в различных статусах:
  - Новые (10%)
  - Просмотренные (20%)
  - В работе (30%)
  - На ревью (15%)
  - Выполненные (20%)
  - Отмененные (5%)
- Приоритеты: низкий, средний, высокий, критический
- Подзадачи (40% задач имеют 2-5 подзадач)
- Комментарии к задачам
- Чек-листы с выполненными/невыполненными пунктами
- Множественные исполнители
- История изменений (activity log)
- Дедлайны и прогресс выполнения

### 📊 Опросы
- Типы опросов:
  - Одиночный выбор
  - Множественный выбор
- Статусы: активные (60%), завершенные (40%)
- Голоса участников (40-90% охват)
- Комментарии к опросам
- Анонимные и публичные голосования
- Видимость: публичные, отдельные, для приглашенных

### 📅 Календарь
- Различные типы событий:
  - Встречи с участниками
  - Дедлайны
  - Личные события
- События в прошлом и будущем (от -30 до +60 дней)
- Повторяющиеся события (15%)
- Участники со статусами (принято, отклонено, возможно, ожидает)
- Целодневные события (10%)
- Местоположение (конференц-залы, zoom, и т.д.)

## Использование

### Базовая команда (загрузить все данные)

```bash
# Из корня проекта
go run scripts/seed/main.go --all

# Или с очисткой базы данных
go run scripts/seed/main.go --all --clean
```

### Выборочная загрузка

Вы можете загрузить только определенные типы данных:

```bash
# Только пользователи
go run scripts/seed/main.go --users

# Только чаты
go run scripts/seed/main.go --chats

# Только задачи
go run scripts/seed/main.go --tasks

# Только опросы
go run scripts/seed/main.go --polls

# Только календарь
go run scripts/seed/main.go --calendar

# Комбинация
go run scripts/seed/main.go --users --chats --tasks
```

### Настройка количества записей

```bash
go run scripts/seed/main.go --all \
  --user-count 100 \
  --chat-count 50 \
  --task-count 200 \
  --poll-count 30 \
  --event-count 150
```

### Очистка базы данных

```bash
# Очистить базу перед загрузкой
go run scripts/seed/main.go --all --clean
```

## Флаги команды

| Флаг | Описание | Значение по умолчанию |
|------|----------|----------------------|
| `--all` | Загрузить все типы данных | false |
| `--clean` | Очистить базу данных перед загрузкой | false |
| `--users` | Загрузить пользователей и отделы | false |
| `--chats` | Загрузить чаты и сообщения | false |
| `--tasks` | Загрузить задачи | false |
| `--polls` | Загрузить опросы | false |
| `--calendar` | Загрузить календарные события | false |
| `--user-count` | Количество пользователей | 50 |
| `--chat-count` | Количество чатов | 30 |
| `--task-count` | Количество задач | 100 |
| `--poll-count` | Количество опросов | 20 |
| `--event-count` | Количество событий | 80 |

## Переменные окружения

Убедитесь, что в вашем `.env` файле установлена переменная `DATABASE_URL`:

```env
DATABASE_URL=postgres://user:password@localhost:5432/taxionback?sslmode=disable
```

## Особенности

### Реалистичные данные
- Используется библиотека `gofakeit` для генерации имен, email, описаний
- Русские имена, названия отделов и должностей
- Временные метки распределены реалистично
- Взаимосвязи между сущностями (например, чаты для задач)

### Безопасность
- Все пароли хешируются с использованием bcrypt
- Дефолтный пароль для всех пользователей: `password123`
- Суперадмин создается автоматически

### Распределение данных
- Статусы и приоритеты распределены согласно реальной статистике
- Различные типы сообщений, задач, опросов, событий
- Прогресс задач соответствует их статусу

### Связи между данными
- Задачи могут иметь связанные чаты
- События календаря могут быть связаны с задачами
- Пользователи распределены по отделам и подразделениям
- Иерархия задач (родительские/дочерние)

## Решение проблем

### Ошибка подключения к базе данных
Убедитесь, что:
1. PostgreSQL запущен
2. DATABASE_URL правильно настроен в `.env`
3. База данных существует
4. У пользователя есть права на создание таблиц

### Конфликты уникальности
Используйте флаг `--clean` для очистки базы перед загрузкой

### Медленная загрузка
Уменьшите количество записей с помощью флагов `--*-count`

## Дополнительная разработка

Для добавления новых типов данных:

1. Создайте новый файл в `seeders/`
2. Реализуйте функцию `Seed*`
3. Добавьте вызов в `main.go`
4. Добавьте соответствующие флаги

Пример:
```go
func SeedNotifications(db *database.DB, count int) ([]*notificationModels.Notification, error) {
    // Ваша логика генерации
}
```
