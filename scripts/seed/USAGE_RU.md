# Инструкция по использованию генератора моковых данных

## Что будет создано

Генератор создаст реалистичные данные для всех сервисов:

- **50 пользователей** в 8 отделах с подразделениями
- **30 чатов** (групповые и личные) с сообщениями, реакциями, статусами прочтения
- **100 задач** в разных статусах (новые, в работе, на ревью, выполненные)
- **20 опросов** (одиночный и множественный выбор) с голосами
- **80 календарных событий** (встречи, дедлайны, личные)

## Подготовка

1. Установите зависимости:
```bash
go get github.com/brianvoe/gofakeit/v7
go get github.com/joho/godotenv
```

2. Проверьте `.env` файл (должен быть в корне проекта):
```env
DATABASE_URL=postgres://user:password@localhost:5432/taxionback?sslmode=disable
```

## Запуск

### Простой способ (все данные с очисткой БД)

```bash
go run scripts/seed/main.go --all --clean
```

### С настройкой количества

```bash
go run scripts/seed/main.go --all --clean \
  --user-count 30 \
  --chat-count 20 \
  --task-count 50 \
  --poll-count 10 \
  --event-count 40
```

### Загрузка только определенных данных

```bash
# Только пользователи
go run scripts/seed/main.go --users

# Пользователи и чаты
go run scripts/seed/main.go --users --chats

# Все кроме календаря
go run scripts/seed/main.go --users --chats --tasks --polls
```

## Доступные флаги

| Флаг | Описание |
|------|----------|
| `--all` | Загрузить все типы данных |
| `--clean` | Очистить БД перед загрузкой |
| `--users` | Только пользователи |
| `--chats` | Только чаты |
| `--tasks` | Только задачи |
| `--polls` | Только опросы |
| `--calendar` | Только календарь |
| `--user-count N` | Количество пользователей (по умолчанию 50) |
| `--chat-count N` | Количество чатов (по умолчанию 30) |
| `--task-count N` | Количество задач (по умолчанию 100) |
| `--poll-count N` | Количество опросов (по умолчанию 20) |
| `--event-count N` | Количество событий (по умолчанию 80) |

## Детали генерации

### Пользователи
- Русские имена, фамилии, отчества
- Реальные должности
- 8 отделов: Разработка, Маркетинг, Продажи, Бухгалтерия, HR, IT, Поддержка, Юридический
- Руководители отделов автоматически назначаются

### Чаты
- 60% групповые (3-10 участников)
- 40% личные (2 участника)
- Сообщения: текст (80%), изображения (10%), файлы (5%), системные (5%)
- Реакции (👍, ❤️, 😊, 👏, 🔥, ✅)
- Статусы прочтения для всех участников

### Задачи
- Распределение статусов: новые 10%, просмотренные 20%, в работе 30%, на ревью 15%, выполненные 20%, отмененные 5%
- Подзадачи (40% задач)
- Комментарии (60% задач)
- Чек-листы (30% задач)
- Прогресс соответствует статусу

### Опросы
- Типы: одиночный выбор, множественный выбор
- 60% активные, 40% завершенные
- 40-90% пользователей голосуют
- Комментарии к опросам (40% вероятность)

### Календарь
- Встречи (с участниками)
- Дедлайны
- Личные события
- События от -30 до +60 дней от текущей даты
- 15% повторяющиеся события
- Участники с ответами (принято, отклонено, возможно, ожидает)

## Возможные проблемы

**Ошибка подключения к БД:**
- Проверьте, что PostgreSQL запущен
- Проверьте DATABASE_URL в .env
- Проверьте права пользователя БД

**Ошибки уникальности:**
- Используйте флаг `--clean` для очистки БД перед загрузкой

**Медленная загрузка:**
- Уменьшите количество записей через флаги `--*-count`
- Загружайте данные по частям без флага `--all`
